# Concert Ticket Booking Backend

## Purpose

Backend engineering assessment for GEEK Up.

The goal is NOT to build a production-ready system.

The goal is to demonstrate backend engineering thinking, software architecture, code organization, and trade-off decisions.

---

## Business Context

Online Concert Ticket Booking Platform.

Customer Features

- Browse concerts
- View ticket categories
- Reserve tickets
- Apply vouchers
- Track booking status

Operation Features

- Manage concerts
- Manage ticket inventory
- Monitor bookings
- Validate ticket availability
- Manage vouchers
- Update booking status

---

## Technical Goals

Priority:

1. Clean Architecture
2. Modular codebase
3. Transaction correctness
4. Overselling prevention
5. Duplicate booking prevention
6. Documentation
7. Tests

NOT priority:

- Fancy UI
- Production deployment
- Payment gateway
- Notifications

---

## Tech Stack

- NestJS
- Prisma
- PostgreSQL
- Docker
- Swagger
- Jest
- Redis (optional)

---

## Architecture

Modular Monolith

NOT Microservices

Reason:

- Assessment duration is only 48 hours.
- Easier to reason.
- Easier to test.
- Easier to maintain.
